ARCHITECTURE SUMMARY
================================================================================

This project uses the following critical architectural decisions:

1. LOCAL JS FILES IN /js FOLDER
   - GSAP 3.13.1 (gsap.min.js)
   - ScrollTrigger (ScrollTrigger.min.js)
   - SplitText (SplitText.min.js)
   - Lenis (lenis.min.js)

   ALL animation libraries are served locally from the /js folder to bypass
   CDN blocking issues (network filters, ISPs, browser extensions). These files
   are installed via NPM and copied from node_modules to ensure reliability.

2. TAILWIND COMPATIBILITY BRIDGE
   - The tailwind.config.js contains a "Compatibility Bridge" that maps legacy
     class names (text-dark, text-light, bg-paper-light, teal-dark, etc.) to
     the new SOTD "Paper & Teal" color palette.
   - This ensures old components work with the new design system without
     breaking existing code.

3. DECOUPLED SCROLL & ANIMATION ENGINES
   - Lenis (smooth scroll) runs independently using requestAnimationFrame()
   - GSAP animations are separate and include safety checks
   - If GSAP fails to load, scroll still works and content remains visible
   - This fault-tolerant architecture prevents the "deadlock" bug where
     scroll freezes if animations crash

4. SPLITTEXT MASKING SYSTEM
   - Hero text uses GSAP SplitText with custom CSS classes (.line-parent,
     .line-child) for masked reveal animations
   - The .line-parent provides overflow masking
   - The .line-child slides up with a "back.out(1.2)" bounce easing

5. COLOR PALETTE: "Paper & Teal"
   - Primary: Deep Teal (#0F4C4C - ink-primary)
   - Accent: Vermilion Red (#C44536 - ink-vermilion)
   - Background: Warm Off-White (#FDFBF7 - paper-base)
   - Secondary: Aged Paper (#F2EFE5 - paper-aged)

        const heroH1 = document.querySelector(".hero-text h1");
        const heroLead = document.querySelector(".hero-text .lead");
        const heroCta = document.querySelector(".hero-cta");
        const heroLetter = document.querySelector(".hero-letter");

        if (heroH1) {
           console.log("‚úÖ Animating hero section");

           const tl = gsap.timeline({ defaults: { ease: "power3.out" } });

           tl.fromTo(heroH1,
              { opacity: 0, y: 50 },
              { opacity: 1, y: 0, duration: 1.2, delay: 0.2 }
           );

           if (heroLead) {
             tl.fromTo(heroLead,
               { opacity: 0, y: 30 },
               { opacity: 1, y: 0, duration: 1 },
               "-=0.8"
             );
           }

           if (heroCta) {
             tl.fromTo(heroCta,
               { opacity: 0, y: 30 },
               { opacity: 1, y: 0, duration: 1 },
               "-=0.7"
             );
           }

           if (heroLetter) {
             tl.fromTo(heroLetter,
               { scale: 0.8, rotation: -15, opacity: 0 },
               { scale: 1, rotation: 0, opacity: 1, duration: 1.4, ease: "elastic.out(1, 0.5)" },
               "-=1.0"
             );
           }
        }

        // 4. Scroll-triggered animations for cards
        const cards = gsap.utils.toArray(".card, .ai-mode-card, .tech-badge, .pricing-card");
        console.log("‚úÖ Found " + cards.length + " cards");

        if (cards.length > 0) {
          gsap.set(cards, { opacity: 0, y: 50 });

          ScrollTrigger.batch(cards, {
            start: "top 85%",
            onEnter: (batch) => {
              gsap.to(batch, {
                opacity: 1,
                y: 0,
                stagger: 0.15,
                duration: 1,
                ease: "power3.out"
              });
            }
          });
        }

        console.log("‚úÖ All animations initialized!");
      });
    </script>
